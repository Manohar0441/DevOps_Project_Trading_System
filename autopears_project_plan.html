1 Docker Containerisation

Week 1–2Docker · AWS ECR · docker-compose
Goal: Every service runs inside a reproducible, minimal Docker image. Extend the existing docker-compose.dev.yml to cover all 8 services and publish production-grade images to ECR.

Image Strategy

  • Base image: python:3.11-slim — minimise attack surface
  • Multi-stage builds: builder stage installs deps, final stage copies artefacts only
  • Non-root USER (appuser) in all images — required by EKS pod security policies
  • One Dockerfile per service: services/<name>/Dockerfile
  • Shared base image in ECR: <account>.dkr.ecr.<region>.amazonaws.com/autopears/base

Dockerfile Template (apply to all 8 services)

Dockerfile
# ── Stage 1: builder ─────────────────────────────────────
FROM python:3.11-slim AS builder
WORKDIR /app
COPY services/common ./common
COPY services/<service_name>/requirements.txt .
RUN pip install --no-cache-dir --prefix=/install -r requirements.txt

# ── Stage 2: production image ─────────────────────────────
FROM python:3.11-slim
WORKDIR /app
RUN addgroup --system appgroup && \
    adduser --system --ingroup appgroup appuser
COPY --from=builder /install /usr/local
COPY services/common ./common
COPY services/<service_name> ./<service_name>
USER appuser
EXPOSE 8000
CMD ["python", "-m", "<service_name>.main"]

4 Jenkins CI/CD Pipeline

Week 7–8Jenkins · Jenkinsfile · ECR · K8s rolling deploy
Goal: Every git push triggers infra/jenkins/Jenkinsfile — running tests, building images, scanning with docker scout, pushing to ECR, and deploying to the matching K8s environment automatically.

Branch → Environment Mapping

feature/* → autopears-dev (EKS dev) Auto on push
develop → autopears-staging (EKS staging) Auto on push
main → autopears-prod (EKS prod) Manual approval gate

Jenkinsfile

Groovy — infra/jenkins/Jenkinsfile
pipeline {
  agent { label 'docker-agent' }

  environment {
    ECR_REGISTRY = credentials('ecr-registry-url')
    AWS_REGION   = 'us-east-1'
    SERVICES     = 'screening scoring risk portfolio batch notification frontend'
  }

  stages {

    stage('Checkout') {
      steps { checkout scm }
    }

    stage('Unit Tests') {
      steps {
        sh 'pip install -r requirements-test.txt'
        sh 'pytest services/ --junitxml=reports/tests.xml -v \
            --cov=services --cov-report=xml'
      }
      post {
        always {
          junit      'reports/tests.xml'
          cobertura  coberturaReportFile: 'coverage.xml'
        }
      }
    }

    stage('Build & Scan Images') {
      steps {
        script {
          env.SERVICES.split(' ').each { svc ->
            sh """
              docker build \\
                -t ${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER} \\
                -f services/${svc}_service/Dockerfile .
              docker scout cves ${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER} --exit-code
            """
          }
        }
      }
    }

    stage('Push to ECR') {
      steps {
        withAWS(credentials: 'aws-jenkins-role', region: env.AWS_REGION) {
          sh 'aws ecr get-login-password | docker login --username AWS --password-stdin $ECR_REGISTRY'
          script {
            env.SERVICES.split(' ').each { svc ->
              sh "docker push ${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER}"
              sh "docker tag  ${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER} \
                              ${ECR_REGISTRY}/autopears/${svc}:latest"
              sh "docker push ${ECR_REGISTRY}/autopears/${svc}:latest"
            }
          }
        }
      }
    }

    stage('Deploy → Dev') {
      when { branch 'feature/*' }
      steps {
        script {
          env.SERVICES.split(' ').each { svc ->
            sh """
              kubectl set image deployment/${svc}-service \
                ${svc}-service=${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER} \
                -n autopears-dev
              kubectl rollout status deployment/${svc}-service \
                -n autopears-dev --timeout=120s
            """
          }
        }
      }
    }

    stage('Deploy → Staging') {
      when { branch 'develop' }
      steps {
        script {
          env.SERVICES.split(' ').each { svc ->
            sh """
              kubectl set image deployment/${svc}-service \
                ${svc}-service=${ECR_REGISTRY}/autopears/${svc}:${BUILD_NUMBER} \
                -n autopears-staging
              kubectl rollout status deployment/${svc}-service \
                -n autopears-staging --timeout=120s
            """
          }
        }
      }
    }

    stage('Prod Approval') {
      when { branch 'main' }
      steps {
        input message: "Deploy build ${BUILD_NUMBER} to PRODUCTION?",
              ok: 'Deploy',
              submitterParameter: 'APPROVED_BY'
      }
    }

    stage('Deploy → Prod') {
      when { branch 'main' }
      steps {
        sh 'kubectl apply -f infra/k8s/apps.yaml -n autopears-prod'
        script {
          env.SERVICES.split(' ').each { svc ->
            sh """
              kubectl rollout status deployment/${svc}-service \
                -n autopears-prod --timeout=300s
            """
          }
        }
      }
    }
  }

  post {
    success {
      slackSend channel: '#deployments', color: 'good',
        message: "Build ${BUILD_NUMBER} deployed to ${BRANCH_NAME}"
    }
    failure {
      slackSend channel: '#alerts', color: 'danger',
        message: "Build ${BUILD_NUMBER} FAILED on ${BRANCH_NAME} — ${BUILD_URL}"
    }
  }
}

Jenkins Agent on EKS

  • Deploy Jenkins controller via official Helm chart into namespace jenkins
  • Kubernetes plugin — agents spin up as ephemeral pods, terminate after each run
  • Agent pod spec includes Docker-in-Docker (dind) sidecar for image builds
  • Jenkins ServiceAccount IAM role: ecr:GetAuthorizationToken, ecr:PutImage, eks:DescribeCluster
  • All credentials stored in Jenkins Credentials Store — never in Jenkinsfile or Git
  • Shared library: infra/jenkins/vars/ for reusable pipeline step functions

5 Monitoring, Security & Go-Live

Week 9–10CloudWatch · X-Ray · Secrets Manager · Load testing

Observability Stack

  • CloudWatch Container Insights — CPU, memory, network per pod and node
  • Fluent Bit DaemonSet — ships all pod stdout/stderr to CloudWatch Logs
  • CloudWatch Alarms — p99 latency > 2s, error rate > 1%, pod restarts > 3
  • AWS X-Ray SDK — distributed tracing across screening → scoring → risk calls
  • S3 Access Logs + WAF logs to CloudWatch for security audit trail

CloudWatch Alarm Example

HCL — modules/eks/alarms.tf
resource "aws_cloudwatch_metric_alarm" "pod_restarts" {
  alarm_name          = "autopears-pod-restarts-${var.environment}"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = 2
  metric_name         = "pod_number_of_container_restarts"
  namespace           = "ContainerInsights"
  period              = 60
  statistic           = "Sum"
  threshold           = 3
  alarm_actions       = [aws_sns_topic.alerts.arn]
  dimensions = {
    ClusterName = var.cluster_name
    Namespace   = "autopears-prod"
  }
}

Secrets Management Rules
